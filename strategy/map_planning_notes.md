# Map Planning Notes

Last War S6のCpt Hedgeインタラクティブ地図を、JDX/#534の作戦検討に使うためのメモ。

確認日: 2026-05-03
情報源:

- https://cpt-hedge.com/maps/season-6/interactive
- ユーザー添付画像: シーズンマップ/グループ/ルール/中心エリア/祭壇

## 使える機能

地図ツール側で確認できた主な機能:

- 複数の計画グリッド作成
- 連盟ごとの色分けとエリア割り当て
- 都市、拠点、漁場、祭壇の表示
- 影響値、占領上限、バフの確認
- 進軍ルートを番号付きで置く `Proposed Path Mode`
- JSONによるエクスポート/インポート

作戦資料としては、スクリーンショットだけでなく、地図上に色と番号を置いた計画をJSONで共有できる点が大きい。

## 地図ゾーン対応

Cpt HedgeのS6地図は3000x3000座標で、外周8ゾーンと中央ゾーンに分かれている。

| ゾーン | 座標範囲 | 対応サーバー | 陣営 | JDX視点 |
|---|---|---|---|---|
| A | x0-1000, y0-1000 | #534 | 密林 | 自サーバー。左上。JDXの主戦域 |
| H | x1000-2000, y0-1000 | #509 | 密林 | 上中央。#534が直接接する味方候補 |
| G | x2000-3000, y0-1000 | #503 | 湿地 | 右上。#509方面の主敵候補 |
| B | x0-1000, y1000-2000 | #476 | 湿地 | 左中段。#534が直接接する最大圧力 |
| I | x1000-2000, y1000-2000 | #8061 | 中央 | 大河氏族。祭壇、中央漁場、Ancestral Temple |
| F | x2000-3000, y1000-2000 | #480 | 湿地 | 右中段。敵中段 |
| C | x0-1000, y2000-3000 | #523 | 湿地 | 左下。敵下段 |
| D | x1000-2000, y2000-3000 | #511 | 密林 | 下中央。味方だが支援が必要になりやすい |
| E | x2000-3000, y2000-3000 | #440 | 密林 | 右下。味方上位集中型。全体方針の軸 |

外周A-Hは各221エリア。内訳は概ね、漁場120、聖域1、村50、兵舎20、集会場12、Warzone Outpost 4、Trade Post 10、Console 4。中央Iは397エリアで、Ancestral Temple、漁場、Snake/Echo/Gust/Feather/Treehaven Altarが含まれる。

ゲーム内画像で確認した陣営:

- 密林陣営: #440 / #509 / #511 / #534
- 湿地陣営: #476 / #480 / #503 / #523

#534が初期に直接接するのは、味方#509と敵#476。#440は味方方針の軸だが、地図上は右下で#534から遠い。したがって、#440方針を待ちながら、#534自身は#476を受け止め、#509方面を候補線として維持する。

## 初期集合案

現時点のおすすめは、#534内の中央から東寄りを主集合、南側を少数監視にする配置。

### 主集合

候補: A61、A155-A188、A49-A85周辺
座標感: x450-825、y350-650付近

理由:

- A61のLv7聖域と中央のLv3/4漁場に近い
- A166/A167/A176/A177のWarzone Outpost周辺を押さえやすい
- 南の#476側に寄りすぎず、東の#509方面にも展開余地を残せる
- Week 1の内政、漁場、都市確保を優先しやすい

### #509方面の観測線

候補: A151/A161/A171/A181/A191/A201
座標感: x925、y225-725の東側Trade Post列

ここは正式な盟約前提の進軍線ではなく、#440方針と#509側の意図が見えるまでの候補線として扱う。主力を置きすぎず、地図上では「接続候補」「観測」「後日判断」として色を分ける。

### #476方面の警戒線

候補: A192-A221
座標感: y725-925の南側列

ここは主集合ではなく、少数の監視・遅滞用。#476へ深追いするためではなく、敵の初動、漁場圧力、接続点への接近を早く見るための配置にする。

### 中央・祭壇

中央Iは別グリッドで管理する。祭壇は隣接不要で参加できる可能性が高い一方、中央漁場・都市・前哨基地・破壊ルートは接続条件が絡むため、Week 1の本隊集合と混ぜて考えない。

## 地図グリッドの作り方

幹部で共有するなら、Cpt Hedge上で以下のグリッドを分けると見やすい。

| グリッド名 | 目的 |
|---|---|
| `JDX Week1 Internal` | #534内の主集合、漁場、都市、Outpostを整理 |
| `476 Watch Line` | 南側の監視・遅滞候補を整理 |
| `509 Candidate Route` | #440方針待ちの#509接続候補を仮置き |
| `Altar Candidates` | 中央Iの祭壇候補と担当を整理 |
| `Outpost Candidates` | 前哨基地候補を複数案で比較 |

## エクスポート運用

地図のエクスポートはJSONで、概ね以下の情報を持つ。

```json
{
  "version": "1.0",
  "exportDate": "YYYY-MM-DD...",
  "grids": [
    {
      "name": "Season 6 - JDX Week1 Internal",
      "alliances": [
        {
          "id": "jdx",
          "name": "JDX",
          "areaIds": ["A61", "A155"],
          "color": "#8BC1F7"
        }
      ],
      "pathSteps": [
        {
          "areaId": "A171",
          "allianceId": "jdx",
          "step": 1
        }
      ]
    }
  ]
}
```

Discord共有は画像でよいが、幹部間の修正はJSONの方がよい。誰かが地図上で修正したら、エクスポートJSONを保存して差し替える。

## 次に必要な情報

- #440の全体方針
- #509が実際にどの連盟を前に出すつもりか
- #534内でJDX以外の連盟がどの都市・漁場を担当するか
- ゲーム内開幕後、Aゾーンの実際の都市名/座標がCpt HedgeのIDと一致しているか

## 現時点の結論

この地図は十分活用できる。全体配置は`#534=A`、`#509=H`、`#503=G`、`#476=B`、`中央#8061=I`、`#480=F`、`#523=C`、`#511=D`、`#440=E`として扱う。Week 1内政・南警戒・#509候補・中央祭壇を別グリッドに分ける。JDXの初期集合はA中央から東寄りを本命、南側は少数監視に留めるのが現実的。#440は味方方針の軸だが右下Eなので、初期の#534防衛を#440頼みにしすぎない。
