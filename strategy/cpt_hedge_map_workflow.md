# Cpt Hedge Map Workflow

Last War S6のCpt Hedgeインタラクティブ地図を、JDX/#534幹部の作戦図作成に使うための運用手順。

確認日: 2026-05-03

情報源:

- https://cpt-hedge.com/maps/season-6/interactive
- ゲーム内シーズンマップ画像
- `strategy/map_planning_notes.md`

## 位置づけ

Cpt Hedge地図は、公式ルール確認用ではなく、作戦検討用のホワイトボードとして使う。

| 用途 | 扱い |
|---|---|
| ゾーン配置、都市/拠点/漁場/祭壇の位置確認 | 使う。ただし開幕後にゲーム内表示と照合する |
| 幹部向けの集合場所、警戒線、進軍候補の図示 | 積極的に使う |
| JSONエクスポート/インポートによる案の保存 | 幹部内の作業用として使う |
| 公式ルール、開放日、占領条件、隣接条件の判断 | 使わない。公式Wiki、ゲーム内ルール、実地確認を優先する |
| 敵の意図や盟約判断の確定 | 使わない。外交情報と実際の動きで判断する |

## 管理方針

- 地図編集担当を1-2名に絞る。
- Discord共有は原則スクリーンショットにする。
- JSONは幹部作業用としてローカル保存する。
- GitHubには原則JSONを載せない。載せるのは承認済み方針の要約mdだけにする。
- 個人作成サイトなので、サイト側の仕様変更や表示違いを前提に、重要判断は必ずゲーム内で確認する。

## ローカル保存場所

JSONエクスポートや作業途中スクリーンショットは、以下へ保存する。

```text
map_exports/YYYY-MM-DD/
```

例:

```text
map_exports/2026-05-03/20260503_W1_534_Internal_v01.json
map_exports/2026-05-03/20260503_476_Watch_Line_v01.png
```

`map_exports/` はローカル作業用とし、GitHubには含めない。

## グリッド命名

複数案を比較しやすくするため、グリッド名は日付、目的、版数を入れる。

| グリッド名 | 目的 |
|---|---|
| `20260503_W1_534_Internal_v01` | Week 1の#534内政、主集合、漁場、Outpost整理 |
| `20260503_476_Watch_Line_v01` | #476方面の監視線、遅滞候補、深追い禁止ライン |
| `20260503_509_Candidate_Route_v01` | #509連携候補。#440方針待ちの仮案 |
| `20260503_Central_Altar_v01` | 火曜祭壇候補と担当整理 |
| `20260503_Outpost_Options_v01` | 前哨基地候補の複数案比較 |
| `20260503_Destruction_Route_v01` | Week 3以降の敵都市破壊順序の検討 |

版数は `v01`, `v02` のように増やす。承認済みだけ `approved` を付ける。

例:

```text
20260503_W1_534_Internal_v03_approved
```

## 色分け

同じ色を毎回使い、スクショだけでも意味が通るようにする。

| 対象 | 推奨色 | 意味 |
|---|---|---|
| JDX/#534主集合 | 青緑 | 本隊・内政主軸 |
| #534内の他連盟 | 緑系 | 自サーバー内協力枠 |
| #509候補線 | 水色 | 連携候補。未確定 |
| #440方針軸 | 金色 | 味方陣営の全体方針 |
| #476圧力線 | 赤 | 最大警戒・遅滞対象 |
| #503/#480/#523 | 橙/紫/桃 | その他敵方面 |
| 中央I/大河氏族 | 灰色 | 中央管理、祭壇、漁場 |
| 要確認地点 | 黄色 | ゲーム内照合が必要 |

## 戦力ランキングの重ね方

戦力ランキングは、Cpt Hedge地図上で「どこを厚く見るか」を決める補助情報として使う。

優先する入力:

- `outputs/s6powerrank_alliance_rankings_2026-05-03.csv`
- `outputs/s6powerrank_hero_rankings_2026-05-03.csv`
- `strategy/server_strategy_comparison_2026-05-03.md`

地図への反映:

| 対象 | 地図上の扱い | 理由 |
|---|---|---|
| #476 | 赤を濃く、警戒線を太めにする | Top9/Top6/Top3すべて最大。#534南の最大圧力 |
| #509 | 水色の候補線を太めにする | 味方Top9 2位。#503方面と中央参加の生命線 |
| #440 | 金色で方針軸として示す | Top3は強いが#534から遠い。初期防衛の直接戦力とは分ける |
| #511 | 薄い緑または支援マーク | 味方で最も薄い。単独主戦線より支援対象 |
| #480/#503/#523 | 敵色を分ける | #503は#509方面、#480/#523は中央・下側連動を監視 |

注意:

- Cpt Hedgeには戦力値そのものを大量に書き込まない。Discord用画像が読みにくくなる。
- 戦力差は「線の太さ」「濃い色」「注意マーク」で表す。
- 司令官/英雄名はOCR揺れがあるため、個人名を地図やDiscord指名に使う前に元画像で確認する。

## Week 1用の作り方

1. `20260503_W1_534_Internal_v01` を作る。
2. #534=Aゾーンを基準に、主集合をA中央から東寄りへ置く。
3. 南側A192-A221付近は、監視・遅滞として赤または黄色で薄く示す。
4. #509方面A151/A161/A171/A181/A191/A201付近は、正式ルートではなく候補線として水色で示す。
5. 漁場、都市、Warzone Outpostを優先度別に色分けする。
6. Discord共有用にスクリーンショットを1枚作る。
7. 共有文には「#509線は未確定」「#440方針待ち」「南は深追いしない」を必ず添える。

## #476警戒線の作り方

`476 Watch Line` は、攻撃計画ではなく防衛・遅滞計画として作る。

- 主力集合を南へ寄せすぎない。
- #476側に隣接されると危険な都市、漁場、接続点を目立たせる。
- 早期警戒役、遅滞役、本隊合流地点を分ける。
- 「越えてよい線」と「越えない線」を別色にする。
- Discord共有時は、赤線を攻撃指示と誤解されないようにする。

## #509候補線の作り方

`509 Candidate Route` は、正式な盟約計画ではなく、#440方針後に判断するための材料として作る。

- #509上位連盟がどの方面へ出るかを記録する。
- #503方面を#509が抑える場合、#534が何を担当するかを仮置きする。
- JDX単独で決められない地点は `candidate` 扱いにする。
- 盟約候補、進軍候補、防衛候補を混ぜない。

## 中央・祭壇の作り方

中央Iは、Week 1本隊集合とは別グリッドにする。

- 祭壇は火曜集合用の別働計画として管理する。
- 1連盟3祭壇上限と排他関係を確認する。
- 都市/前哨基地/破壊ルートとは別に考える。
- 祭壇は隣接不要の可能性が高いが、中央の都市・漁場・前哨基地まで隣接不要とは扱わない。

## レビュー手順

幹部共有前に、以下を確認する。

| 確認項目 | 判定 |
|---|---|
| 公式ルールではなくCpt Hedge表示だけに依存していないか | 必須 |
| #440方針待ちの内容を確定案として書いていないか | 必須 |
| #509連携を未確定のまま、JDX行動の前提にしていないか | 必須 |
| 南#476方面へ主力を寄せすぎていないか | 必須 |
| 都市破壊後に隣接が消える前提を考慮しているか | Week 3以降必須 |
| Discordに貼ったとき、色の意味が説明なしでも誤解されにくいか | 必須 |

## 幹部Discordへの貼り方

画像1枚と短文で共有する。

```text
【S6地図案 / Week 1】
目的：#534内政と初期集合の整理

青緑：JDX主集合候補
黄：ゲーム内確認待ち
赤：#476方面の警戒線
水色：#509連携候補。未確定

注意：
これはCpt Hedge地図を使った作戦図です。
公式ルールの根拠ではないため、開幕後にゲーム内表示と照合します。
#440方針と#509側の動きが見えるまでは、南は防衛・遅滞、東は候補線として扱います。
```

## 結論

Cpt Hedge地図は、JDXの作戦立案ではかなり有用。ただし、有用なのは「図にして合意形成する」部分であり、「ルールを確定する」部分ではない。

当面は、Week 1内政、#476警戒、#509候補、中央祭壇、前哨基地候補の5グリッドを作り、Discordにはスクリーンショット、ローカルにはJSON、GitHubには承認済み要約mdという分け方で運用する。
