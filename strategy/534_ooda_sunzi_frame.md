# #534 OODA/Sunzi Decision Frame - 2026-05-19

目的: `OODALOOP2.pdf` と `Sun_Tsu_The_Art_of_Warfare.pdf` を、#534/JDXの実務判断に使うための短縮フレームにする。

このメモは引用集ではなく、#534の現状に合わせた運用メモ。幹部が次の応戦枠までに、何を見て、何を捨て、何を実行するかを決めるために使う。

## 結論

#534の活路は、#476や#503を正面から倒すことではない。

勝ち筋は、敵の進行テンポを崩し、都市隣接と中央進出を遅らせ、#509/#440/#511を含む味方陣営の主戦力が動ける時間を作ること。

JDXの役割は、全部を守る主役ではなく、次の3つに絞る。

1. #534内の都市隣接阻止線を保つ。
2. 中央#8061で#503の進路を遅らせる。
3. 水曜/土曜の応戦枠で、#476側の弱点だけを限定的に突く。

## OODAを#534向けに使う

OODAは、作戦を固定するためではなく、状況変化に合わせて早く回すために使う。

| 段階 | #534で見ること | 出すべき結論 |
|---|---|---|
| Observe | 占領ログ、スクショ、保護残り、安全時間、敵の新規取得、味方の未取得セル | 何が変わったか |
| Orient | #476/#503がどのルートを開けたいか、味方がどこで詰まっているか | どこが本当に危ないか |
| Decide | 守る、捨てる、奪う、交代する、外交する、待つ | 次の応戦枠までの優先順位 |
| Act | 担当者へDM、取得/放棄/再取得、集結、シート更新、スクショ保存 | 実行済み証拠と次の確認時刻 |

重要なのは、1回の会議で完璧な作戦を作ることではない。次の応戦枠までに、敵より早く判断を更新すること。

## 1ループの標準手順

次の応戦枠ごとに、幹部はこの順番で回す。

### 1. Observe

最低限見るもの:

| 項目 | 見る場所 | 判断材料 |
|---|---|---|
| 新規占領ログ | シーズンチャット、スクショ、読取ログ | 敵の進路、味方の抜け、保護開始時刻 |
| 保護切れ | 管理表たたき、公開ダッシュボード | 次に攻撃可能になる地点 |
| 安全時間 | 連盟安全時間、安全時間履歴、ゲーム画面 | 保護計算の前提、変更前後の固定条件 |
| 未所有セル | 全体マップたたき、読取ログ | 敵に無料で取られる穴 |
| 味方稼働 | R4/R5、Discord、ゲーム内オンライン | 実際に動ける連盟 |

この段階では議論しすぎない。事実と推測を分ける。

### 2. Orient

次の質問で、危険度を決める。

| 質問 | はいの場合 |
|---|---|
| 取られると都市隣接につながるか | Cライン扱い。最優先で守る |
| 取られても敵の接続が伸びるだけか | A/Bライン扱い。遅滞・監視でよい |
| #503の中央進路を助けるか | #509/FHXと共有。JDX単独判断にしない |
| 味方が保有しているが安全時間が悪いか | 放棄・譲渡・再取得候補 |
| 相手が強い本隊か、弱い連盟か | 本隊なら避ける。弱い連盟なら限定反撃候補 |

ここで大事なのは、見えている敵を全部敵主力と扱わないこと。守る価値が低いセルまで守ると、主力が先に消耗する。

### 3. Decide

毎回、判断を3つに絞る。

| 優先 | 決めること | 例 |
|---|---|---|
| 1 | 絶対に落とせない地点 | 都市隣接、Cライン、中央の進路封鎖点 |
| 2 | 捨ててもよい地点 | A線、敵の進行確認用、後で取り直せるFG |
| 3 | 反撃する地点 | #476側の弱い連盟、空白、保護切れ、孤立地点 |

決めないことも決める。

- #476本隊とは正面決戦しない。
- 中央#8061をJDX単独で取り切ろうとしない。
- Aライン全域を守らない。
- 未確認座標でDMや外交を進めない。

### 4. Act

実行は短く、証拠を残す。

| 行動 | 担当 | 証拠 |
|---|---|---|
| 取得/放棄/再取得 | 所有連盟R4/R5 | スクショ、取得ログ |
| 交代依頼 | JDX幹部、対象連盟幹部 | DM文面、対象座標 |
| #509/FHX連絡 | 指名担当 | 代わってほしい座標、期限 |
| 476外交 | 1名に限定 | 送った文面、返答 |
| シート更新 | 管理担当 | 読取ログ、管理表たたき |

実行後は、次の確認時刻を必ず決める。実行して終わりにしない。

## 現状の優先順位

### 最優先: 都市隣接阻止

#534にとって最悪なのは、敵に都市隣接の連続ルートを作られること。

Aラインを抜かれること自体は負けではない。Cラインが抜かれて都市隣接が成立することが負けに近い。

判断:

- Aは監視・遅滞線。
- Bは必要筋だけ厚くする。
- CはJDX/KTVS/SHA/n09/89M/4THで守る。
- 下位連盟だけにCを持たせない。

### 次点: 中央#8061の進路妨害

中央はJDXが主役を取る場所ではなく、#509が#503を止めるための補助線。

JDXがやること:

- #503の進路上のFG/拠点を囲う。
- #509/FHXへ交代してほしい地点を座標で渡す。
- #503が中央FGを取って汚染地へ入ったら、強い人で集結を当てて進行を遅らせる。
- 弱い人は穴埋めと交代に回す。

やらないこと:

- JDX単独で中央を取り切ろうとする。
- #509が囲う前に、JDXだけで突出する。
- 保護明けや種別未確認のまま動く。

### 余力: #476側への限定反撃

反撃は必要。ただし、戦線を広げるためではなく、相手のテンポを崩すために使う。

狙う対象:

- 弱い連盟が持つFG。
- 保護切れ直後の孤立セル。
- 取ると敵の接続が一手遅れるセル。
- 取ってもこちらのCラインが薄くならないセル。

狙わない対象:

- #476本隊がすぐ反応できる主線。
- 取っても維持できない深い地点。
- 取得枠と保有上限を浪費する地点。
- 外交上、#509/#440の方針と矛盾する地点。

## 476外交の使い方

476外交は、信頼のためではなく時間稼ぎと意図確認のために使う。

出してよい情報:

- 無駄な消耗は避けたい。
- こちらも状況を見ながら動く。
- 不必要な正面衝突は増やしたくない。

出さない情報:

- A/B/C防衛線の具体配置。
- 交代要員。
- #509/FHXの動き。
- 中央#8061の赤丸地点。
- どの保護切れを狙っているか。

約束しないこと:

- 完全不可侵。
- 固定の境界線。
- #509や#440の行動まで含めた保証。

## 水曜/土曜前チェック

攻撃可能日または応戦枠の前に、これだけ確認する。

| 確認 | OK条件 |
|---|---|
| 対象座標 | ゲーム内スクショまたは読取ログで確認済み |
| 所有連盟 | 管理表たたきとゲーム画面が一致 |
| 保護切れ | `d hh:mm:ss` 表示とシート計算が大きくズレていない |
| 安全時間 | 取得時点の安全時間で固定。変更後の安全時間を過去へ適用しない |
| 取得枠 | 担当連盟に枠が残っている |
| 保有上限 | FG/Stronghold上限に空きがある |
| 接続 | 自連盟または盟約で攻撃可能 |
| 予備 | 失敗時の代替連盟または撤退判断がある |

## 10分判断テンプレ

急に敵が動いた時は、これで判断する。

```text
1. その地点は都市隣接につながるか？
2. Cラインか、Bラインか、Aラインか？
3. いま守ると主力が消耗しすぎないか？
4. 捨てた場合、次に守る地点はどこか？
5. 反撃するなら、敵のどの弱点を突けるか？
6. 連絡すべき相手はJDX内か、#534幹部か、#509か、476外交担当か？
7. 10分後に確認するスクショは何か？
```

この7問に答えられない場合は、主力を出さずに情報を取りに行く。

## 使い方

このファイルは、作戦の正解ではなく判断の型。

スクショ、読取ログ、管理表たたき、全体マップたたきの情報が増えるたびに、次の順で更新する。

1. 新規占領ログを読取ログに入れる。
2. 座標と保護切れを管理表たたきで確認する。
3. A/B/Cまたは中央#8061のどこに影響するかを判断する。
4. 次の応戦枠までの「守る・捨てる・奪う・外交」を決める。
5. 実行後のスクショで再確認する。
