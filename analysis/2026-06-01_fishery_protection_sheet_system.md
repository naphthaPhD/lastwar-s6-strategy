# Last War S6 漁場保護管理システム

## 1. Executive summary

漁場ごとの保護切れ時刻、応戦枠、ワンパン候補、放棄後タイマーを管理する Google Sheets + Apps Script パッケージを追加した。主目的は「次にどの漁場がいつ開くか」を一覧化し、ヘカテ個人の記憶に依存しない防御壁運用へ寄せること。

## 2. Context

入力仕様は、交戦日を水曜・土曜、応戦枠を `15:00-16:00`, `23:00-24:00`, `07:00-08:00`、15時枠を安全期間として扱う前提。実運用で見る最終開放枠は安全時間を除いた `WED_23`, `THU_07`, `SAT_23`, `SUN_07` の4状態に限定する。

2026-06-01時点で、実運用先は `https://docs.google.com/spreadsheets/d/1Zzp53UbwcZdD80BXO7xXYfjPxkUvBYwdIN5_I9KnE80/edit`。#534だけでなく #509/#476 も後から入るため、漁場の識別子は `#534:A-1` のような `位置キー` に変更した。

## 3. Key facts

- Apps Script 本体を `tools/fishery_protection_sheet/Code.gs` に保存した。
- 導入手順と運用方針を `tools/fishery_protection_sheet/README.md` に保存した。
- ゲームルールの整理を `rules/fishery_protection_timer_rules.md` に保存した。
- `漁場一覧`, `イベント一覧`, `シミュレーター`, `カレンダー`, `ライン定義`, `侵攻ルート確認`, `連盟判定`, `盟約管理`, `連盟キャパ管理` の9シートを作成・更新する設計。
- #534 は `A列/B列/C列`、#509 は `A-1, B-1, C-1, D-1` 方向の `1列/3列/5列`、#476 は逆向きの `K列/J列/I列` を防衛ラインとして扱う。
- `NEXT_PROTECTION_END`, `ABANDON_PROTECTION_END`, `CAN_ABANDON`, `PROTECTION_STATE`, `PROTECTION_STATE_NAME`, `NEXT_AFTER_PROTECTION_PUNCH` のカスタム関数も用意した。
- 保護パン後の状態遷移は `WED_23 -> SUN_07`, `THU_07 -> SAT_23`, `SAT_23 -> THU_07`, `SUN_07 -> WED_23`。
- `15:00-16:00` は最終開放枠として採用せず、計算上15時に当たった場合は23時へ補正する。
- `侵攻ルート確認` は、同じ侵攻ルート上で同じ開放枠が連続する場合に `危険`、分散している場合に `分散` と表示する。
- `連盟判定` は敵/味方/同サーバ/他サーバ味方を分けて管理する。初期値として `xJR/xjR`, `476C`, `476B` を敵にした。
- `盟約管理` で盟約状態が `有効` の相手は、保護パン不可として扱う。
- `連盟キャパ管理` は1日取得上限、所有都市、都市由来の漁場保有上限、現所有漁場数を見て制約メモを出す。
- `ワンパン必要` は手動判断を上書きしない。空欄または `自動:` で始まる値だけ自動更新する。

## 5. Interpretation

このシステムは、ワンパンの実行判断を自動化するものではない。23時開放は `自動:候補`、7時開放は `自動:7時要判断` として、実行者が状況判断できる表示に留める。7時側を危険として放置するヘカテ方式にも対応する。

重要なのは、`A列は7時`, `B列は23時` のような固定表現を使わないこと。ラインは侵攻ルート確認のために使うが、保護切れは必ず漁場単位で保持する。

敵味方・盟約・キャパは、保護切れ時刻とは別レイヤーとして管理する。敵が保有するルートは侵攻前提で警戒し、味方や盟約中の相手には保護パンを不可または要判断に落とす。

## 6. Risks

1. 放棄後保護の実ゲーム挙動は、実画面で追加確認が必要。
2. Apps Script のタイムゾーンが `Asia/Tokyo` でないと、日時表示がずれる可能性がある。
3. 実ゲーム画面の保護終了表示と表計算が食い違う場合は、画面表示を優先する必要がある。

## 7. Recommended actions

1. 実運用先 Google Sheet の Apps Script を `tools/fishery_protection_sheet/Code.gs` に更新する。
2. #509/#476 のスクショ投入時は、先に `ライン定義` の軸と方向に沿って `防衛ライン` と `位置キー` を埋める。
3. `連盟判定` に未登録連盟を追加し、敵/味方/同サーバ/他サーバ味方を更新する。
4. `盟約管理` に有効盟約を入れて、保護パン不可の誤判定を防ぐ。
5. `連盟キャパ管理` に1日取得上限、所有都市、都市由来の漁場上限を入れる。
6. `侵攻ルート確認` で敵保有かつ同じ開放枠が連続しているルートを優先確認する。
7. 実画面の保護終了表示と計算結果を数件突き合わせる。
8. 放棄を1件テストできるタイミングで、放棄後保護終了が Apps Script の仮定と一致するか確認する。
