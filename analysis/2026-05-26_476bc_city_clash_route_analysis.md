# 476B / 476C City Clash Route Analysis

## 1. Executive summary

2026-05-27の都市争奪戦では、#476B/#476Cが既に#534内に持つ漁場だけで、#534側の複数都市を破壊候補にできる。

最重要リスクは、都市そのものの固定防衛ではなく、#476B/#476Cに `D-9` / `D-11` / `E-7` 方面の追加漁場を取られ、右方向・#503側への接続準備を進められることである。

明日の#534側の優先順位は以下。

1. `D-11` を476Bに取らせない。
2. `D-9` を476B/476Cに取らせない。
3. `c-12(JDX)`、`c-10(SHA)`、`c-8(SHA)`、`b-10(SHA)` の破壊警戒を置く。
4. 476C上方向、476B右方向の分担を前提に監視班と即応班を分ける。

## 6. Split-operation scenario

ユーザー指摘の「片方の敵が上に来て、もう片方が右の漁場を攻めて#503側への侵攻を進める」シナリオは十分現実的。

もっとも自然な分担は以下。

| 役割 | 敵連盟 | 主な動き | #534への影響 |
|---|---|---|---|
| 上・内側圧力 | 476C | A/B/C-7、C-9から b-6/b-8/c-8/c-10 方面へ圧力 | SHA/MOE側を混乱させ、防衛人員を上へ吸う |
| 右方向・#503側準備 | 476B | C-11/D-7から D-9/D-11/E-7 方面へ拡大 | JDX/SHA都市を脅しつつ、土曜以降の右方向接続を作る |

この分担をされた場合、#534が一番崩れやすいのは「都市防衛に全員を寄せた結果、D-11またはD-9を取られる」形である。

## 7. Risks

1. `D-11` を取られると、`c-12(JDX)` に加えて `d-12(JDX)` が開き、JDX都市を二面で見られる。
2. `D-9` を取られると、`d-10(SHA)` が開き、476B/476Cの右方向展開が現実化する。
3. 476Cが上側を叩き、476Bが右側を伸ばすと、防衛指示が都市防衛班と漁場即応班に割れて混乱しやすい。
4. 都市は即再取得前提で扱いにくい。破壊されると接続・勢力値・保有枠に影響が出るため、漁場より不可逆性が高い。
5. 空欄都市 `c-6 / d-6` は管理表上は都市だが、所有状態と実際の攻撃可否をゲーム内で確認する必要がある。

## 8. Recommended actions

### 最優先防衛

| 優先 | 対象 | 理由 |
|---|---|---|
| 1 | D-11 | 476Bの右方向展開とJDX二面圧力を同時に許すため |
| 2 | D-9 | 476B/476Cどちらにも使われる右折入口 |
| 3 | E-7 | 右下へ深く入られ、土曜以降の#503側接続準備になる |
| 4 | c-12(JDX) | 既存476B足場から直接危険 |
| 5 | c-10/c-8/b-10(SHA) | 476B/476C双方から重複して狙われやすい |

### 明日の指揮運用

1. 476C監視班: `A-7 / B-7 / C-7 / C-9` 周辺を見る。
2. 476B監視班: `C-11 / D-7 / D-9 / D-11 / E-7` 周辺を見る。
3. 都市防衛班: `c-12 / c-10 / c-8 / b-10` を中心に、防衛対象を2-3都市に絞る。
4. 即応班: 敵が `D-11` または `D-9` に入ったら都市より漁場を優先して止める。
5. Capture管理: こちらも漁場枠を使い切らず、取り返し役と防衛役を分ける。
