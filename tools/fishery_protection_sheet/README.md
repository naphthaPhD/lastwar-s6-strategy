# Last War S6 漁場保護管理システム

Google Sheets に貼り付ける Apps Script 一式。目的は「どの漁場が次にいつ開放されるか」を管理者が一覧で確認できる状態にすること。

## シート構成

`setupFisheryProtectionWorkbook()` を実行すると、次の11シートを作成する。

| シート | 用途 |
|---|---|
| `00_目次` | 日常運用、入力マスタ、取込確認、旧/自動生成タブの分類と見る順番 |
| `漁場一覧` | エリア、位置キー、所有者、防衛ライン、座標、保護切れ、ワンパン担当、最終操作日時を管理する主表 |
| `イベント一覧` | 次回保護切れ日時順の時系列一覧 |
| `シミュレーター` | 取得、保護パン、放棄の日時を入れて次回保護切れを試算する |
| `カレンダー` | 安全時間を除いた4つの開放枠ごとの対象漁場一覧 |
| `ライン定義` | #534/#509/#476 など、エリアごとのライン軸とスクショ投入方向を管理する補助表 |
| `侵攻ルート確認` | 侵攻方向ごとに連続漁場の開放枠を並べ、同じ枠が連続する場合に警告する |
| `選択漁場ビュー` | 漁場行、開放枠、侵攻ルートを選択した時に、対象漁場と保護パン候補連盟を表示する |
| `管理表たたき参照` | 元の `#534 / 管理表たたき` から値だけをコピーする参照タブ |
| `連盟判定` | 連盟タグごとの敵/味方/中立/同サーバ/他サーバ味方などを管理するマスタ |
| `盟約管理` | 盟約相手を管理し、有効な盟約相手への保護パンを不可にする |
| `連盟キャパ管理` | 1日取得上限、所有都市、都市由来の漁場保有上限を管理する |

## 主な計算ルール

- 交戦日は水曜と土曜。
- 応戦枠は `07:00-08:00`, `15:00-16:00`, `23:00-24:00`。
- `15:00-16:00` は安全期間として、保護終了候補になった場合は `23:00-24:00` へ繰り下げる。
- 実運用の最終開放枠は `WED_23`（水曜23時）、`THU_07`（木曜7時）、`SAT_23`（土曜23時）、`SUN_07`（日曜7時）の4状態だけで管理する。
- 通常取得、保護パンは「次の宣戦日の同時刻より1個後ろの応戦時間」まで保護する。
- 放棄は、放棄時刻から見た次の応戦枠を基準に1個後ろの応戦時間まで放棄後保護に入る。
- 放棄した連盟は24時間再取得不可。
- 応戦開始1時間前から応戦終了までを放棄不可として表示する。
- 保護パン後の状態遷移は `WED_23 -> SUN_07`, `THU_07 -> SAT_23`, `SAT_23 -> THU_07`, `SUN_07 -> WED_23` として関数化している。
- `連盟判定` で `敵` とした連盟は、侵攻してくる前提で `侵攻ルート確認` に警戒表示する。初期値では `xJR/xjR`, `476C`, `476B` を敵として登録する。
- `盟約管理` で盟約状態が `有効` の相手は、敵味方にかかわらず保護パン不可として扱う。
- `連盟キャパ管理` で、1日取得上限や都市由来の漁場上限が不足する場合は制約メモに表示する。

## 導入手順

1. Google Sheets で対象スプレッドシートを開く。
2. `拡張機能` -> `Apps Script` を開く。
3. `Code.gs` の中身を `tools/fishery_protection_sheet/Code.gs` で置き換える。
4. Apps Script のタイムゾーンを `Asia/Tokyo` に設定する。
5. `setupFisheryProtectionWorkbook()` を実行して権限を承認する。
6. シート側メニュー `漁場保護` -> `一覧を再計算` を使う。

## 入力の基本

`漁場一覧` のうち、まず手入力する列は以下。

- `漁場名`
- `所有連盟`
- `エリア`
- `位置キー`
- `防衛ライン`
- `ライン順序`
- `ライン方向`
- `座標`
- `現在保護切れ日時`
- `ワンパン役`
- `備考`
- `最終操作日時`
- `最終操作`

`最終操作` は `取得`, `保護パン`, `放棄` のいずれか。`最終操作日時` が入っている場合は、この日時から次回保護切れを再計算する。未入力の場合は `現在保護切れ日時` をそのままイベント一覧へ反映する。

`位置キー` は `#534:A-1` のように `エリア:漁場名` で持つ。#534は従来どおり `A列/B列/C列`、#509は `A-1, B-1, C-1, D-1` のように同じ数字をラインとして扱うため `1列/3列/5列`、#476は逆向きに `K列/J列/I列` を使う。

`表示順` は隠し列。Google Sheets の文字列ソートでは `A-1` の次が `A-11` になるため、`番号部分 -> ライン順序` の数値キーを使って `A-1, B-1, C-1, A-3, B-3, C-3...` の順に並べる。

`状態ラベル`, `状態表示`, `保護パン後の次回保護切れ`, `安全時間補正後` は自動計算列。`現在保護切れ日時` が15時枠の場合は、最終開放枠として扱わず23時枠へ補正する。

スクショ投入時の標準フォルダ例は以下。

- `漁場スクショ/534/ABC`
- `漁場スクショ/509/ABC`
- `漁場スクショ/476/KJI`

## 運用メモ

- 通常は `00_目次` から入り、日常運用は `漁場一覧4枠`, `開放カレンダー4枠`, `侵攻ルート確認`, `選択漁場ビュー`, `連盟安全期間`, `シミュレーター` を見る。
- `選択漁場ビュー` は `onSelectionChange(e)` で軽量更新する。対象は `漁場一覧4枠`, `漁場一覧`, `開放カレンダー4枠`, `カレンダー`, `侵攻ルート確認` だけに限定し、通常編集時の負荷を抑える。
- `漁場一覧4枠` の行を選ぶとその漁場、`開放カレンダー4枠` の枠を選ぶと同じ開放枠の漁場、`侵攻ルート確認` の行を選ぶと同じ侵攻ルート上の漁場を表示する。
- 保護パン候補連盟は `連盟判定` で味方/同サーバ味方として扱える連盟から表示する。`連盟キャパ管理` の1日取得上限または都市由来余力が不足している連盟は候補から外す。所有者が味方、盟約中、未登録の場合は候補ではなく不可/要確認を表示する。
- `管理表たたき参照` は `漁場保護` -> `管理表たたき値コピー` で更新する。コピー元は `#534` スプレッドシートの `管理表たたき`。元シートには書き込まず、値だけをコピーし、書式・入力規則・数式はコピーしない。
- `イベント一覧` と `カレンダー` は旧/自動生成ビューとして残す。4枠運用では `開放カレンダー4枠` を優先する。
- `ワンパン必要` は、空欄または `自動:` で始まる場合だけ自動更新する。`必要`, `不要`, `見送り`, `要確認` などの手動判断は上書きしない。
- 23時開放は `自動:候補`、7時開放は `自動:7時要判断` とする。ヘカテ方式のように7時側を放置する判断があるため、システムは候補表示に留める。
- `A列は7時`, `B列は23時` のような固定表現は使わない。必ず漁場単位で次回開放枠を持つ。
- `侵攻ルート確認` では、同じ侵攻ルート上で同じ開放枠が隣接していれば `危険`、交互または分散していれば `分散` と表示する。
- 敵味方や盟約は固定値にしない。状況が変わったら `連盟判定` と `盟約管理` を更新し、再計算する。
- 実ゲーム画面の保護終了表示と食い違う場合は、画面表示を優先し、備考に差分を残す。
